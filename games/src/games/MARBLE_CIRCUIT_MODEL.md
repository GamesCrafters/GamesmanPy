# Marble Circuit 建模说明（无 UI）

## 1. 物理游戏抽象成数据

### 1.1 棋盘 (Board)

- **槽位 (slots)**：底板上的格子，块块只能放在这些位置。用一维编号 `0, 1, 2, ...`。**实物为 10 个槽位。**
- **入口 / 出口**：**实物为 8 个入口**（8 颗球）、**5 个出口**（底部）；胜负为各出口进球个数符合题目要求。
- **拓扑**：每块有 **四个口**（不是四条边）：**左上、右上、右下、左下**，编号 `0,1,2,3`。相邻槽位通过「口对口」相连；部分口为**入口**或**出口**。
- **在代码里**：用「槽位数量」+「邻接表」表示：
  - `adj[slot][口]` = `(邻居槽位, 邻居的哪个口)` 或 `('entry', entry_id)` 或 `('exit', exit_id)`。
  - 口编号：0=左上(TL)、1=右上(TR)、2=右下(BR)、3=左下(BL)。

例如 2×2 的 4 个槽位（简化示例）：

```
  [entry0] [entry1]
     slot0   slot1
     slot2   slot3
  [exit0]  [exit1]
```

- slot0 上边 = entry0，下边 = slot2 的上边，左边 = 边界，右边 = slot1 的左边。
- 实现时用一张表写死或从配置文件读入即可，不需要画图。

### 1.2 块块 (Blocks)

实物共 **10 块**：青绿 3、黄 2、橘 2、玫红 3。每一类块用**路径**描述，不用画图：

- **四个口**：块有 4 个口，编号 0=左上、1=右上、2=右下、3=左下。
- **路径**：块内部是「从哪个口进、从哪个口出」的通道（口对口）。
  - **玫红（四通）**：四个口互通。
  - **橘 T**：左上、右上 都通 右下（左下堵）。
  - **黄 T**：左上、右上 都通 左下（右下堵）。
  - **青绿（菱形 / )( 形）**：**两条独立轨道**——左上↔左下、右上↔右下。
  - **实物规则：块块不可旋转**。每种块只有一种固定朝向。

### 1.3 挑战 (Challenge)

一个关卡 = 固定信息，不随“一步棋”变化：

- **初始摆放**：部分槽位已有块（类型 + 朝向），其余为空。
- **待放置的块**：手里还有哪些块（按类型计数），例如 2 个直条、1 个 L。
- **目标**：若干条「从入口 i 放珠，必须进出口 j」，例如 `[(entry0, exit1), (entry1, exit0)]`。

## 2. 游戏状态（供求解器 / TUI 用）

- **局面 (position)**：当前棋盘上每个槽位是什么（空 或 块类型）+ 手里还剩哪些块（不可旋转，无朝向）。
- **一步 (move)**：在某个空槽位放一块块（块型固定朝向）。move 可编码为 `(slot_id, block_type)` 或压成一个 int。
- **终局**：所有该关卡的块都放完了（没有空槽可放，或手里没块了）。此时用「模拟滚珠」判断输赢：
  - 对每个 (entry_i, exit_j) 目标：从 entry_i 出发，沿当前板子的通道走，看是否到达 exit_j；
  - 全部到达 → 赢；否则 → 输。

不需要 UI：状态、步法、胜负都只用整数和上述数据结构算出来。

## 3. 滚珠模拟（无 UI 的核心）

- 把当前板子建成一张**有向图**：节点 = (槽位, 边) 或 入口/出口 id。
- 根据每块块的路径（固定朝向），在相邻槽位之间连边（例如 slot0 的边 2 连到 slot2 的边 0）。
- 对每个目标 (entry_i, exit_j)：从 entry_i 做 BFS/DFS，看能否到达 exit_j。
- 全部能到达 → 该局面 Win，否则 Loss。

## 4. 状态与步法的整数编码（给 GamesmanPy）

- 每个槽位：0 = 空，1～4 = 块类型（青绿/橘/黄/玫红），不可旋转故无朝向维。
- 棋盘部分：多槽位拼成多进制数（base = 5，即 0 空 + 4 种块）。
- 手里块数：用额外整数表示（如 4 个数：青绿/橘/黄/玫红各剩几块）。
- 一步 move：`slot_id * 4 + block_type`（在合法时唯一）。

这样 `position` 和 `move` 都是 int，适合 `start()`、`generate_moves()`、`do_move()`、`primitive()` 等接口。

## 5. 小结

| 概念       | 无 UI 的表示方式                          |
|------------|-------------------------------------------|
| 棋盘       | 槽位数量 + 邻接表（槽位-边 ↔ 邻居或 entry/exit） |
| 块块       | 路径列表 (in_side, out_side)，不可旋转、固定朝向 |
| 挑战       | 初始摆放 + 待放置块 + 各出口目标个数（8 球进 5 出口） |
| 局面       | 每槽 空/块型 + 剩余块数（4 种）           |
| 一步       | 在某个空槽放某块（无朝向选择）             |
| 胜负判断   | 块放满后，按图模拟滚珠，检查是否全部达标 |
